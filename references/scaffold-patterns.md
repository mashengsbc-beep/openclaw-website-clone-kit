# Scaffold Patterns

When the user wants actual files, create a run folder like:

`/home/node/OpenClawBox/deliveries/website-clone-runs/<slug>/`

Recommended contents:

- `README.md` — short run summary
- `inspection/` — screenshots, text dumps, notes
- `spec/website-rebuild-spec.md` — the main rebuild brief
- `implementation/` — optional code scaffold

If creating a code scaffold, keep it minimal and honest:
- basic package manifest
- page shell
- component folders
- asset folders
- content/data file

Do not generate a huge fake codebase unless the user asked for actual implementation depth.
