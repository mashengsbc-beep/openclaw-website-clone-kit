# Evaluation

Judge website clone requests on these axes:

## 1. Surface complexity
- simple landing page
- multi-section marketing site
- content site/blog
- authenticated product app
- animation-heavy experience

## 2. Extraction difficulty
- mostly static HTML/text/images
- heavy responsive changes
- hidden/tabbed content
- video/canvas/WebGL-driven visuals
- data loaded after auth or API calls

## 3. Rebuild fit in OpenClaw
Best fit:
- landing pages
- brochure/brand sites
- campaign pages
- simple homepages

Medium fit:
- blog/article pages
- documentation fronts
- moderate interactivity

Weak fit:
- dashboards
- authenticated SaaS apps
- complex multi-step products
- pages whose value is in backend logic rather than presentation

## Verdict labels
Use one clear verdict:
- good candidate for local rebuild
- good candidate for partial reconstruction
- learn-from-only, not ideal for direct cloning
- avoid
